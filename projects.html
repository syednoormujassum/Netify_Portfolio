🤖 GST ChatBot

A Retrieval-Augmented Generation (RAG) powered intelligent chatbot for Goods and Services Tax queries

📝 Project Overview

The GST ChatBot is a sophisticated application designed to help users understand complex Goods and Services Tax (GST) regulations. Dedicated to my beloved father, this project leverages cutting-edge AI technologies to provide accurate, context-aware answers about GST regulations without the need for manual document searching.

✨ Key Features

  • Intelligent document processing and PDF text extraction
  • Smart chunking with overlap for better context retention
  • Semantic search using SentenceTransformer embeddings
  • RAG pipeline for accurate, context-aware responses
  • Clean Streamlit web interface for easy interaction
  • Source attribution showing document references and confidence scores
  • Real-time response generation using Groq API
  • Vector database storage with ChromaDB

🏗️ Architecture

The application follows a modern RAG architecture: PDF Documents → Text Extraction → Chunking → Embeddings → Vector Store → Query Processing → Response Generation

🛠️ Technology Stack

Streamlit
Python
Groq API
ChromaDB
SentenceTransformer
PyMuPDF
Langchain
NumPy/Pandas

📊 Core Components

  • Data Loader: Loads and processes PDF documents from GST sources
  • Chunking Manager: Intelligently splits documents with overlap for context
  • Vector Store: Manages ChromaDB for efficient vector storage and retrieval
  • RAG Retriever: Handles query processing and response generation
  • Streamlit App: User-friendly web interface for interaction

🎯 Use Cases

  • Quick lookup of GST regulations and procedures
  • Understanding tax implications for different scenarios
  • Business compliance and filing assistance
  • Educational resource for GST learners

📚 RAG Pipeline

A complete Retrieval-Augmented Generation pipeline with advanced document processing and semantic search

📝 Project Overview

This project implements a complete RAG (Retrieval-Augmented Generation) pipeline from scratch, demonstrating best practices in document processing, semantic embeddings, and intelligent retrieval. It serves as a foundation for building AI applications that require access to external knowledge bases.

✨ Key Features

  • PDF document parsing using PyMuPDF with text extraction
  • Recursive text splitting for optimal chunk sizing
  • SentenceTransformer-based semantic embeddings
  • ChromaDB for scalable vector storage with unique ID management
  • Cosine similarity-based retrieval using sklearn
  • LLM integration for generating contextual responses
  • Full context preservation with chunking overlap

🔄 Pipeline Flow

Document → PDF Parsing → Text Extraction → Recursive Chunking → Embedding Generation → Vector Storage → Similarity Search → Context Assembly → LLM Processing → Response Generation

🛠️ Technology Stack

Python
PyMuPDF
SentenceTransformer
ChromaDB
Scikit-learn
Langchain
NumPy
ChatGroq

📖 Components

  • Document Parser: PyMuPDF for robust PDF text extraction
  • Chunking Strategy: Recursive text splitter with configurable overlap
  • Embedding Engine: SentenceTransformer for semantic representation
  • Vector Database: ChromaDB with UUID management for unique records
  • Retrieval Engine: Cosine similarity search from sklearn
  • Response Generator: LLM integration for coherent answers

📊 Performance Metrics

  • Fast semantic search with cosine similarity
  • Efficient memory usage with chunked processing
  • Scalable vector storage with ChromaDB
  • Context-aware response generation
